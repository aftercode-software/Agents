# Formatos de post

## Formato A: Situacion - aprendizaje - reflexion

Usar para experiencias reales.

Estructura:

1. Situacion concreta.
2. Que problema o tension aparecio.
3. Que aprendio Matias o Aftercode.
4. Reflexion aplicable a otras empresas.
5. Pregunta o CTA suave.

## Formato B: Problema frecuente - mirada practica

Usar para educar sin sonar academico.

Estructura:

1. Dolor comun de la audiencia.
2. Por que suele pasar.
3. Que alternativa existe.
4. Ejemplo simple.
5. Cierre reflexivo.

## Formato C: Opinion con matiz

Usar para tomar postura.

Estructura:

1. Frase inicial con opinion clara.
2. Matiz: cuando si, cuando no.
3. Ejemplo de negocio.
4. Conclusion breve.
5. Pregunta para abrir conversacion.

## Formato D: Novedad actual traducida a negocio

Usar para tendencias de IA/software.

Estructura:

1. Novedad verificada.
2. Por que importa.
3. Que oportunidad o riesgo abre para empresas.
4. Aplicacion concreta.
5. Reflexion final.

Siempre crear `FUENTES.md`.

## Formato E: Mini caso sin revelar cliente

Usar para prueba social cuidada.

Estructura:

1. Contexto anonimo.
2. Problema.
3. Decision tomada.
4. Resultado o aprendizaje, solo si esta confirmado.
5. Cierre sobre criterio de trabajo.

## Estilo de redaccion

- Hook corto y concreto.
- Parrafos breves.
- Una idea por post.
- No mas de 3 hashtags salvo que el usuario pida otra cosa.
- Evitar emojis salvo que encajen naturalmente.
- Preferir cierres con pregunta humana antes que CTA de venta.
